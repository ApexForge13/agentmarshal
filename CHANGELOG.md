- `feat(voice): inbound voice agent scaffold with Vapi webhook integration, ElevenLabs TTS via Vapi voice config, OpenAI Whisper transcription via Vapi transcriber config, state-transition-triggered mid-call Marshal evaluation. Demonstrates consent-revocation arc: caller revokes recording consent mid-call, Voice agent's continued record_call attempt is caught by Marshal, signed Compliance Receipt emitted, call recovers gracefully. Upgrades voice_recording_consent_state_resolved from Bubble 3 stub to real composite. Sample agent in agentmarshal repo for hackathon demo; production architecture has voice agent in echo-os (documented in lib/voice/README.md).`
- `feat(verify): public Compliance Receipt + Internal Audit verification tool at /verify. Paste any receipt JSON, the tool verifies the Ed25519 signature against AgentMarshal's published public key (also served at /api/verify/public-key as JWK + hex) and surfaces the receipt's structured fields. Reuses the same JCS canonicalization (RFC 8785) and Ed25519 library as receipt emission — no new crypto. Includes pre-generated example receipts (valid Compliance Receipt, valid Internal Audit, tampered receipt). Demonstrates AgentMarshal's actual differentiator: third-party-verifiable, cryptographically-attestable agent decisions — independently checkable without trusting Marshal at all.`
- `feat(timestamp): RFC 3161 external timestamp anchoring for signed receipts via FreeTSA. Every Compliance Receipt and Internal Audit envelope now carries an optional TSA-signed timestamp token proving WHEN the receipt was issued, independent of AgentMarshal's clock or records. Verifier reports timestamp validity alongside signature validity at /verify and via /api/verify/receipt. Degrades gracefully if TSA is unreachable (receipt still signed, just not externally timestamped). Single TSA for v0.2 hackathon; redundancy roadmap documented in lib/compliance/timestamp/README.md. Directly addresses the failure mode in OFAC v. TradeStation Securities (Mar 2026, $1.1M settlement): when regulators examine systems years after the fact, every receipt now has third-party-anchored proof of issuance time, making the 'we have to reconstruct what happened from logs' failure mode structurally impossible.`
- `feat(benchmark): audit-trail tampering scenario suite (Bubble 12) — 5 new scenarios testing the audit-evidence layer (signed receipts, hash chains, RFC 3161 anchors, engine-independent verification). Threat model: operator adversary faking their own audit trail. AgentMarshal catches 5/5; Cedar and OPA score 0/5 structurally (no equivalent artifact in their data model — see Spike G). Adds verifyChain (lib/verify/verify-chain.ts) for sequence linkage verification, and a timestamp/issued_at cross-check (5-minute tolerance) inside verifyReceipt to catch backdated receipts. Deterministic fixture generation via scripts/generate-audit-trail-fixtures.mts (seed-derived forged Ed25519 key; replayed FreeTSA token), with a Section 2 report matrix contrasting AgentMarshal vs Cedar vs OPA on capabilities structurally beyond general-purpose policy engines. Includes a Bubble 11 fixture re-capture to align issued_at with the cryptographic genTime — a necessary precondition for the backdating cross-check; production code already signs-then-stamps within milliseconds, so the prior fixtures had a ~22.5h artificial gap that the new check would have flagged as a false-positive.`

### Changed
- TCPA composite predicate names normalized to predicate-attribute convention to match CAN-SPAM and produce naturally-readable receipt entries (`tcpa_quiet_hours_check` → `tcpa_quiet_hours_respected`, etc.; full map in this commit). Behavior unchanged.
- `lib/compliance/predicates/tcpa/index.ts` refactored to export `registerAllTcpaComposites()` alongside the existing side-effect import. Bare-import behavior unchanged.
- `chore(spec): replace "Day 6" relative references with "Bright Data integration day" semantic anchor across predicate stub comments (tcpa, canspam), agents.md (§2.1, §2.2, §7.2), and compliance-receipt schema + spec doc`
- `refactor(tests): sourcing predicate tests refactored from 1-block to 3-block pattern for parity with operational/voice/sms convention - codebase test-shape consistency, no behavior change`
- `fix(spec): extend scope-contract.schema.json ScopeRule definition to include composite_checks (was implemented in lib/authzen/evaluate.ts + types/authzen.ts since Day 3 Bubble 2 but missing from the normative schema; partial closure of schema-verification Issue queue). New CompositeCheck $def with { predicate: string, input: unknown } shape matching impl.`
- `feat(authzen): EvalContext gains optional action_properties, populated by evaluateRequest from the AuthZEN request's action.properties. Lets composites whose verdict depends on request-time state the contract cannot know at issuance (e.g. mid-call consent) read it without setContractOverride. Additive + optional; existing composites unaffected. First consumer: voice_recording_consent_state_resolved (Bubble 9).`
